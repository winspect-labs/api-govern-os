# Product OS

A structured product knowledge base for teams that use AI agents alongside human engineers and product managers. Product OS consolidates vision, features, backlog, architecture decisions, and research into a single repository—so product knowledge and code stay in sync.

## Quick Links

- [Product Overview](/product) — Vision, pitch, value proposition
- [Features](/features) — Feature status dashboard
- [Architecture](/architecture) — System overview and repo map
- [Backlog](/backlog) — Prioritized backlog and roadmap
- [Research](/research) — Market research and competitor analysis
- [Decisions](/decisions) — Product and architecture decision records
- [Metrics](/metrics) — KPIs and benchmarks

## Get Started

1. Fork this repo
2. Replace example content with your product data
3. Update `data/schema.yaml`, `data/features/*.yaml`, `data/backlog.yaml`, and other data files
4. Add agent rules to your code repositories (see [Developer Workflow](/about-product-os/developer-workflow))
5. (Optional) Enable GitHub OAuth for private deployments
