# Pitch

## One-Liner

Winspect is the API governance layer that gives platform teams a single inventory, subscription-based authorization, and AI-powered discovery—without replacing your authentication or gateways.

## Problem

In large organizations, APIs sprawl across teams, clusters, and repos. There is no single inventory. Authorization is scattered across gateways, custom middleware, and tribal knowledge. Engineers ask "where's the spec?" and get pointed to three different tools. Product managers cannot answer "what APIs do we have?" without archaeology.

## Solution

A governance layer that consolidates API knowledge into one place. Teams subscribe to APIs and get automatic authorization. Kubernetes discovery populates the catalog without manual registration. Semantic search lets engineers find APIs by natural language. The platform augments—does not replace—existing auth and infrastructure.

## Traction

- Core API catalog, subscriptions, and ABAC shipped
- K8s auto-discovery in progress (runtime records, bulk import, manual mapping)
- RAG semantic search in progress
- Subscription-based authorization designed; phased rollout planned
