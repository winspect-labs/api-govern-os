# API Governance Market

## Summary

The API management market has evolved from gateway-centric (Kong, Apigee) to governance-centric. Organizations with 50+ internal APIs face sprawl: no single inventory, authorization scattered, slow onboarding.

## Key Trends

- **Governance layer** — Separate from gateways. Catalog, subscriptions, and authorization flow from a central source.
- **Subscription-based access** — Teams subscribe to APIs; approvers grant; no custom middleware per API.
- **Discovery** — Manual registration is a bottleneck. K8s discovery and semantic search reduce friction.
- **Agent-accessible** — Product knowledge in structured repos; agents read before implementing.

## Implications for Winspect

We position as the governance layer—not a gateway replacement. Augment existing auth and infrastructure. Target platform teams running Kubernetes with many internal APIs.
